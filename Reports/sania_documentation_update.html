Sania Project - Architecture & Workflow Updates

This document details the architectural and workflow enhancements for the Sania project, introducing the "Conversational Brain" and "Agentic AI Assistant".

1. Architecture Modifications

Goal: Transform the system from a standard application to an advanced Agentic AI system by integrating a Conversational Brain in the Backend and a Voice Interface in the Frontend.

Layer 1: Frontend (Interfaces)

Change: Addition of "Avatar Interface" module.

  • Components: Mobile App (Flutter) and Digital Twin (Unity).
  • Functionality: Integration of STT (Speech-to-Text) to listen to the farmer and TTS (Text-to-Speech) for the avatar's response.
  • Justification: enhances accessibility for farmers with lower literacy levels by enabling voice interaction instead of typing.

Layer 2: Backend (API & Business Logic)

MAJOR CHANGE: Introduction of the "AI Orchestrator".

This component differentiates the system from a basic chatbot, acting as the intelligent core.

  • Component A: LLM Engine (e.g., GPT-4o, Llama 3 via API): Serves as the linguistic engine for understanding and generation.
  • Component B: LangChain Agent (The Orchestrator): The decision-making unit that directs actions. It has access to specific Tools:
    • Tool 1 (Text-to-SQL): Interrogates the TimescaleDB (e.g., "What is the current humidity?").
    • Tool 2 (RAG - Knowledge Base): Reads agronomic PDF documents (e.g., "How to treat mildew?").
    • Tool 3 (IoT Actions): Activates valves/actuators via the MQTT API.

Layer 4: Storage

Change: Addition of Vector Store.

  • Implementation: Vector Store or pgvector extension on existing PostgreSQL.
  • Purpose: To store the "memory" of the avatar and technical documents (RAG) as mathematical vectors for efficient retrieval.

2. Workflow Modifications

New Workflow: "The Avatar Interaction Loop"

This workflow describes the sequence of interactions from the user's voice command to the system's action and response.

Step-by-Step Sequence

  1. User Input (Voice): The farmer asks: "Sania, lance l'irrigation sur la zone 2 car le sol est sec."
  2. Transcription (STT): The mobile app converts the audio into text.
  3. Reasoning (LangChain): The Agent receives the text and enters a Reasoning Loop:
    • Is this a knowledge question? No.
    • Is this an action? Yes.
    • Do I need to check something first? Yes, verify the humidity sensor.
  4. Tool Execution (Action):
    • Agent queries TimescaleDB -> Result: "Humidity 18% (Dry)".
    • Agent calls API POST /irrigate/zone2.
  5. Generation (LLM): The Agent generates a natural response: "C'est fait. J'ai vérifié le capteur, il indique 18%. J'ai démarré l'arrosage pour 20 minutes."
  6. Avatar Animation (TTS + Unity/Flutter): The avatar animates and speaks the response.
